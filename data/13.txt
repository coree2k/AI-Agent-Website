## 시스템 개요

이 가이드에서는 매월 자동으로 최신 한국 부동산 정책을 심층 리서치하여 구조화된 데이터로 저장하고, 이메일로 받아볼 수 있는 시스템을 구축합니다.

### 워크플로우 구조

![워크플로우 구조](./n8n_Source/13/2.%20WF.png)

1. **스케줄 트리거 (매월)** → 2. **AI Agent 부동산 정책 리서치** → 3. **Airtable 결과 저장** → 4. **Gmail 메일 자동 발송**

### 주요 기능

- 매월 1회 자동 실행
- Perplexity Deep Research를 활용한 심층 리서치
- AI Agent가 자율적으로 한국 부동산 정책 분석
- 3~5개의 핵심 정책 변화 자동 추출
- 매수/매도 방향성 분석
- 구조화된 JSON 형식 출력 (headline, research, sources)
- Airtable에 리서치 결과 자동 저장
- Gmail로 리서치 결과 자동 발송
- Buffer Window Memory로 이전 리서치 기억

## 사전 준비사항

- n8n 설치 및 실행 (Self-hosted 또는 Cloud)
- OpenAI API 키 (GPT-4.1-mini 사용)
- Perplexity API 키 (Deep Research 기능 사용)
- Airtable 계정 및 Personal Access Token
- Gmail 계정 및 OAuth2 인증

## 문제 해결

### 자주 발생하는 문제들

#### 1. Perplexity API 할당량 초과

**증상**: Research 실패, API limit 에러
**해결책**:
- Perplexity 대시보드에서 크레딧 확인
- 월간 실행으로 제한하여 사용량 절약
- 필요시 Pro 플랜으로 업그레이드

#### 2. JSON 파싱 에러

**증상**: Structured Output Parser에서 에러 발생
**해결책**:
- System Message에서 JSON 형식 출력 강조:
```
CRITICAL: Return ONLY valid JSON. No markdown, no explanations, just JSON.
```
- Temperature를 낮춰서 일관성 향상 (0.3-0.5 권장)
- Output Parser 스키마가 정확한지 확인

#### 3. 메모리 컨텍스트 충돌

**증상**: 이전 리서치 내용이 현재 결과에 혼재
**해결책**:
- Session Key를 월별로 변경:
```
session_key: research_{{ $now.format('YYYY-MM') }}
```
- 또는 매 실행 시 메모리 초기화

#### 4. Airtable 필드 매핑 오류

**증상**: 데이터가 Airtable에 저장되지 않음
**해결책**:
- Airtable 테이블 스키마 확인
- 필드명이 정확히 일치하는지 확인
- `{{ $json.output.headline }}` 경로가 올바른지 확인

## FAQ

### Q1. 왜 매월 1회만 실행하나요?
**A**: 부동산 정책은 빠르게 변하지 않으므로 매월 1회면 충분합니다. 더 자주 실행하면 API 비용이 증가하고 중복 정보가 많아집니다.

### Q2. GPT-4.1-mini와 GPT-4의 차이는?
**A**: GPT-4.1-mini는 GPT-4 대비:
- 비용이 약 1/10 수준
- 속도가 2-3배 빠름
- 리서치 요약 작업에 충분한 성능
- 복잡한 추론이 필요하면 GPT-4 사용 권장

### Q3. Perplexity Deep Research가 뭔가요?
**A**: Perplexity의 sonar-deep-research 모델은:
- 여러 웹 소스를 자동으로 검색
- 교차 검증된 정보 제공
- URL 출처 자동 포함
- 일반 검색보다 정확하고 신뢰도 높음

### Q4. Airtable 대신 다른 DB를 사용할 수 있나요?
**A**: 네, 다음으로 대체 가능합니다:
- Google Sheets (무료, 쉬운 연동)
- Notion Database (시각화 우수)
- PostgreSQL (대용량 데이터)
- MongoDB (유연한 스키마)

### Q5. 이메일 형식을 HTML로 바꿀 수 있나요?
**A**: 네, Gmail 노드에서:
1. Options > Format 선택
2. HTML로 변경
3. Markdown을 HTML로 변환하는 Code 노드 추가

### Q6. 여러 사람에게 발송하려면?
**A**: Send To 필드에 쉼표로 구분:
```
email1@example.com, email2@example.com, email3@example.com
```

### Q7. 비용은 얼마나 드나요?
**A**: 월간 예상 비용:
